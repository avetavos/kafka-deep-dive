## The idea in one sentence

The classic exactly-once use case is **consume-transform-produce** â€” read from an input topic, transform, write to an output topic â€” and Kafka makes it exactly-once by committing the **input offsets inside the same transaction** as the output records.

## Why offsets must join the transaction

Consider reading `orders`, enriching each, and writing to `orders-enriched`. If you commit the output in a transaction but commit the input offset separately, a crash between the two either reprocesses (duplicates the output) or skips (loses input). The fix: commit the consumer\'s offsets **as part of the producer\'s transaction** with `sendOffsetsToTransaction()`. Now output records and input progress commit or abort **together**.

```java
producer.initTransactions();
consumer.subscribe(List.of("orders"));

while (running) {
    ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(200));
    if (records.isEmpty()) continue;

    producer.beginTransaction();
    for (ConsumerRecord<String, String> record : records) {
        producer.send(new ProducerRecord<>("orders-enriched", record.key(), enrich(record.value())));
    }
    // Commit the input offsets in the SAME transaction as the output
    Map<TopicPartition, OffsetAndMetadata> offsets = offsetsFor(records);
    producer.sendOffsetsToTransaction(offsets, consumer.groupMetadata());

    producer.commitTransaction();   // outputs + offsets are atomic
}
```

## Handling failures correctly

Two exception classes matter. `ProducerFencedException` and `OutOfOrderSequenceException` are **unrecoverable** â€” another instance has taken over your `transactional.id`; shut down. A general `KafkaException` inside the transaction should **abort and retry** from the last committed offsets:

```java
} catch (ProducerFencedException | OutOfOrderSequenceException e) {
    shutdown();                       // a zombie was fenced; give up this instance
} catch (KafkaException e) {
    producer.abortTransaction();      // roll back and retry the batch
}
```
