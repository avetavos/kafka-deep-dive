## ไอเดียในหนึ่งประโยค

รัน Kafka 4.x บนเครื่องคือการสตาร์ท **broker ตัวเดียวในโหมด KRaft** ไม่มี ZooKeeper ไม่มี dependency ภายนอก แล้วคุยกับมันทั้งหมดผ่าน `--bootstrap-server localhost:9092` เพื่อสร้าง topic, produce record และ consume มันกลับมา

## ทางที่เร็วที่สุด: Docker container เดียว

Image ทางการ `apache/kafka` มาพร้อม broker ที่รองรับ KRaft ซึ่ง format storage ให้ตัวเองและสตาร์ทเป็น broker รวมกับ controller คำสั่งเดียวได้ cluster ที่ใช้งานได้เลย

```bash
# Start a single-node Kafka 4.x broker in KRaft mode, port 9092 exposed
docker run -d --name kafka -p 9092:9092 apache/kafka:latest
```

นั่นคือ cluster ทั้งหมด ไม่มีอะไรต้องติดตั้งเพิ่ม **ไม่มี process ZooKeeper** ใน Kafka 4.x container มี CLI script ติดมาให้ในตัว คุณจึงรันคำสั่ง admin ข้างในด้วย `docker exec` ได้ หรือจะติดตั้ง Kafka tarball บนเครื่องแล้วชี้ script ไปที่ `localhost:9092` ก็ได้

## ทางแบบ manual: format storage แล้วค่อยสตาร์ท

ถ้าคุณรัน tarball ที่โหลดมาแทน Docker KRaft ต้องมีขั้นตอน **format storage** ครั้งเดียว คุณ generate cluster UUID แล้วเขียนมันลง log directory ก่อนสตาร์ทครั้งแรก

```bash
# 1. Generate a unique cluster ID
KAFKA_CLUSTER_ID="$(bin/kafka-storage.sh random-uuid)"

# 2. Format the log directory for KRaft using that ID and the KRaft config
bin/kafka-storage.sh format --standalone \
  -t "$KAFKA_CLUSTER_ID" \
  -c config/server.properties

# 3. Start the broker (it is also the controller in a single-node setup)
bin/kafka-server-start.sh config/server.properties
```

คู่ `random-uuid` กับ `format` คือพิธี bootstrap ของ KRaft มันแทนที่การ handshake กับ ZooKeeper แบบเดิม ทำแค่ครั้งเดียวต่อ data directory หนึ่งอัน

## สร้าง topic, produce, consume

พอ broker ขึ้นแล้ว ทุกคำสั่งเล็งไปที่ `--bootstrap-server` flag `--zookeeper` ไม่มีอยู่แล้ว สร้าง topic ก่อน

```bash
# Create a topic named "demo" with 1 partition (fine for local)
kafka-topics.sh --bootstrap-server localhost:9092 \
  --create --topic demo --partitions 1 --replication-factor 1
```

จากนั้นเปิด producer ใน terminal หนึ่งแล้วพิมพ์ข้อความ บรรทัดละหนึ่ง record

```bash
# Produce: each line you type becomes a record
kafka-console-producer.sh --bootstrap-server localhost:9092 --topic demo
```

และ consumer ใน terminal อีกอันเพื่ออ่านมันกลับตั้งแต่ต้น

```bash
# Consume: --from-beginning replays every record in the topic
kafka-console-consumer.sh --bootstrap-server localhost:9092 \
  --topic demo --from-beginning
```

<Callout title="อย่าพิมพ์ --zookeeper">
ถ้าคำสั่งหรือ tutorial บอกให้ใส่ `--zookeeper localhost:2181` แสดงว่ามันเก่ากว่า Kafka 4.x ใน Kafka แบบ KRaft-only flag นี้ **หายไปแล้ว** CLI tool ทุกตัว ทั้ง `kafka-topics.sh`, `kafka-console-producer.sh`, `kafka-consumer-groups.sh` ใช้ **`--bootstrap-server`** อย่างเดียวสำหรับระบุ address ของ cluster
</Callout>

<Callout title="สรุปหนึ่งบรรทัด">
Kafka 4.x บนเครื่องคือ **broker KRaft ตัวเดียว** ที่ **ไม่มี ZooKeeper** รัน `apache/kafka` บน **Docker** เพื่อสตาร์ทคำสั่งเดียว หรือ format log directory ด้วย **`kafka-storage.sh format`** แล้วสตาร์ทด้วย **`kafka-server-start.sh`** จากนั้นสร้าง produce และ consume ผ่าน **`kafka-topics.sh`**, **`kafka-console-producer.sh`** และ **`kafka-console-consumer.sh`** โดยใช้ **`--bootstrap-server localhost:9092`** เสมอ
</Callout>
