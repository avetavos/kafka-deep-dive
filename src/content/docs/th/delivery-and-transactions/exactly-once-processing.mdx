## ไอเดียในหนึ่งประโยค

use case คลาสสิกของ exactly-once คือ **consume-transform-produce** อ่านจาก input topic แปลง แล้วเขียนไป output topic และ Kafka ทำให้มันเป็น exactly-once ด้วยการ commit **input offset ภายใน transaction เดียวกัน** กับ output record

## ทำไม offset ต้องเข้าไปอยู่ใน transaction

ลองนึกถึงการอ่าน `orders` เติมข้อมูลแต่ละตัว แล้วเขียนไป `orders-enriched` ถ้าคุณ commit output ใน transaction แต่ commit input offset แยกต่างหาก การล่มระหว่างสองจังหวะจะทำให้ประมวลผลซ้ำ (output ซ้ำ) หรือข้าม (input หาย) วิธีแก้คือ commit offset ของ consumer **ให้เป็นส่วนหนึ่งของ transaction ของ producer** ด้วย `sendOffsetsToTransaction()` ทีนี้ output record กับความคืบหน้าของ input จะ commit หรือ abort **พร้อมกัน**

```java
producer.initTransactions();
consumer.subscribe(List.of("orders"));

while (running) {
    ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(200));
    if (records.isEmpty()) continue;

    producer.beginTransaction();
    for (ConsumerRecord<String, String> record : records) {
        producer.send(new ProducerRecord<>("orders-enriched", record.key(), enrich(record.value())));
    }
    // Commit the input offsets in the SAME transaction as the output
    Map<TopicPartition, OffsetAndMetadata> offsets = offsetsFor(records);
    producer.sendOffsetsToTransaction(offsets, consumer.groupMetadata());

    producer.commitTransaction();   // outputs + offsets are atomic
}
```

## จัดการ failure ให้ถูก

exception สองกลุ่มสำคัญ `ProducerFencedException` และ `OutOfOrderSequenceException` เป็นแบบ **กู้คืนไม่ได้** อีก instance เข้ามายึด `transactional.id` ของคุณแล้ว ให้ shutdown ส่วน `KafkaException` ทั่วไปภายใน transaction ควร **abort แล้ว retry** จาก offset ที่ commit ล่าสุด

```java
} catch (ProducerFencedException | OutOfOrderSequenceException e) {
    shutdown();                       // a zombie was fenced; give up this instance
} catch (KafkaException e) {
    producer.abortTransaction();      // roll back and retry the batch
}
```

<Callout title="สรุปหนึ่งบรรทัด">
exactly-once **consume-transform-produce** ทำงานด้วยการเอา input offset ของ consumer **เข้าไปอยู่ใน transaction ของ producer** ผ่าน `sendOffsetsToTransaction()` เพื่อให้ output record กับความคืบหน้าของ input commit แบบ atomic มอง `ProducerFencedException` เป็น fatal (zombie ถูก fence) และ `KafkaException` เป็น **abort แล้ว retry** นี่ให้ exactly-once สำหรับ pipeline ที่อยู่ภายใน Kafka
</Callout>
